# CayleyPy_Pancake_sorting

# 🥞 Pancake Sorting: Решение для соревнования CayleyPy

[![Kaggle](https://img.shields.io/badge/Kaggle-Kerby-blue)](https://www.kaggle.com/competitions/CayleyPy-pancake)

Это репозиторий с моим решением для соревнования **CayleyPy Pancake sorting – outperform B.Gates**[4†L5-L6]. Проект посвящен поиску кратчайших последовательностей префиксных инверсий (также известных как "перевороты блинов") для сортировки случайных перестановок – задачи, которая является NP-трудной в общем случае[reference:0].

## 🧩 О задаче

Соревнование ставит перед участниками задачу: найти последовательность переворотов (префиксных инверсий), приводящую заданную перестановку к отсортированному виду (0, 1, 2, ..., N-1). Переворот в данном контексте — это операция, которая берет первые k элементов массива и переворачивает их порядок.

*   **Входные данные**: Набор перестановок (permutation) разной длины, полученных из `test.csv`.
*   **Выходные данные**: Для каждой перестановки необходимо вернуть строку (solution), содержащую последовательность переворотов (например, "R4.R2.R3..."), которая сортирует массив.
*   **Метрика оценки**: Сумма длин решений по всем тестовым случаям. Чем меньше сумма, тем лучше.
*   **Сложность**: Нахождение минимального количества переворотов (оптимального решения) является **NP-трудной задачей**[reference:1], что требует применения эвристик и умных стратегий поиска.

## 🧠 Подход к решению

Поскольку поиск оптимального решения даже для "средних" N (50-100) может занять экспоненциально растущее время, основное внимание уделялось разработке эвристических алгоритмов, способных находить достаточно короткие пути за разумное время. В ходе работы были реализованы и протестированы следующие подходы:

1.  **Жадный алгоритм**: Классический метод, который на каждом шаге ищет наибольший не на своем месте элемент и переворачивает его наверх, а затем на нужную позицию.
2.  **BFS с эвристиками**: Поиск в ширину, который ограничивает пространство поиска узким коридором, используя эвристики, основанные на локальной упорядоченности соседних элементов.
3.  **Двунаправленный поиск**: Одновременный запуск BFS из стартового и целевого состояний до пересечения. В сочетании с эвристиками этот метод позволил значительно сократить время поиска для самых больших перестановок.
4.  **Итеративное углубление**: Алгоритм, который постепенно увеличивает глубину поиска, что гарантирует нахождение оптимального решения, но зачастую дороже по времени.

Финальное решение комбинирует лучшие эвристики для разных размеров `n`. Для небольших `n` (например, до 16) используется поиск, близкий к оптимальному, чтобы минимизировать счет. Для больших `n` (50-100) применяются более быстрые эвристические алгоритмы, которые находят "хорошее" (но не обязательно оптимальное) решение.
